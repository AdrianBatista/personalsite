# Website Vision — Adrian Felipe Nogueira Batista

## Objective

Create a personal website that positions Adrian not simply as a software developer or an electrical engineer, but as a professional who bridges both disciplines.

The website should communicate that software is his primary tool to solve engineering problems.

> **Electrical Engineer specialized in software development for engineering.**

---

# Brand Positioning

## Core Message

Engineering and software are not separate domains—they complement each other.

The site should consistently reinforce the idea that Adrian develops software that improves engineering processes.

Suggested tagline:

> **Engineering better solutions through software.**

Alternative (PT-BR):

> **Transformando processos de engenharia por meio do desenvolvimento de software.**

---

# Design Philosophy

The website should look more like:

- engineering notebook
- software architecture portfolio

- a social media profile

- technical depth
- organization
- professionalism

# Tone of Voice

Professional.

Technical.

Clear.

Objective.

Avoid marketing language.

Avoid buzzwords.

Avoid exaggerated self-promotion.

Write like engineering documentation.

---

# Writing Style

The content should be:

- concise
- informative
- technically accurate
- confident
- accessible

The website should feel like reading documentation written by an experienced engineer.

---

# Core Narrative

The entire website should tell one story.

It is **not**:

> An electrical engineer who also programs.

Nor:

> A software developer with an engineering degree.

Instead:

> A professional who combines Electrical Engineering and Software Engineering to create better engineering solutions.

Everything on the website should reinforce this idea.

---

# Key Differentiator

The competitive advantage is not knowing multiple programming languages.

The competitive advantage is understanding engineering problems deeply enough to build software that engineers actually use.

The website should emphasize impact over technology.

Instead of saying:

"I know Python."

Prefer:

"I develop software that improves engineering workflows."

Instead of listing frameworks, show how they are used to solve technical problems.

---

# Overall Feeling

A visitor should leave the website thinking:

> "This is an engineer who happens to build excellent software—not a software developer who happens to know engineering."
